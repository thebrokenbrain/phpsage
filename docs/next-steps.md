# Próximos pasos

Aunque PHPSage ya ha alcanzado un estado funcional sólido, todavía quedan varias líneas de evolución abiertas que serían razonables en una siguiente iteración.

## Mejoras previstas

- **Mejorar la experiencia de usuario de la web**: refinar la interfaz para que la navegación, la lectura de resultados y la interacción con la capa de IA resulten más fluidas, claras y consistentes.
- **Permitir seleccionar el proyecto PHP desde la propia interfaz**: hacer que el desarrollador pueda elegir desde la web qué proyecto o ruta quiere analizar, sin depender exclusivamente de la CLI o de rutas predefinidas.
- **Permitir añadir reglas de PHPStan**: incorporar una forma controlada de extender o ajustar la configuración del análisis con reglas adicionales según las necesidades de cada proyecto.
- **Mejorar la integración con proyectos PHP grandes**: cargar correctamente el autoloader del proyecto analizado para que PHPStan pueda resolver clases y símbolos dentro del contexto real del código, reduciendo errores derivados de referencias no encontradas o de un contexto incompleto.
- **Reforzar la documentación para perfiles junior**: ampliar y reorganizar la documentación con un enfoque más orientado a desarrolladores que están empezando, de forma que el proyecto resulte más fácil de entender, levantar y extender.

## Enfoque

Estas mejoras no cambian la base ya construida, pero sí marcarían una siguiente fase de maduración del proyecto centrada en tres frentes: usabilidad, capacidad de adaptación a proyectos reales y accesibilidad para nuevos desarrolladores.
